JavaScript Switch Statement

JavaScript Switch Statement: A switch statement allows you to execute different code blocks based on the value of an expression. It's a more concise alternative to multiple if-else statements.
break Keyword: The break keyword is used to exit a loop or a switch statement immediately.
default Keyword: The default keyword in a switch statement specifies the code to be executed if no case matches the expression.
Common Code Blocks: Code blocks within a switch statement are enclosed in curly braces {}. Each block is associated with a specific case, and the code within the block is executed if the case matches.

JavaScript Loops

JavaScript Loops: Loops allow you to repeatedly execute a block of code until a certain condition is met.
for Loop: The for loop is used when the number of iterations is known before entering the loop. It is a pre-test loop, meaning the condition is evaluated before the code block runs.
while Loop: The while loop repeats a block of code as long as a specified condition is true. It is also a pre-test loop, so if the condition is false from the start, the loop won't run at all.
do/while Loop: The do/while loop is similar to the while loop, but the condition is evaluated after the code block runs. This guarantees that the loop will execute at least once, even if the condition is false at the beginning.

JavaScript Break and Continue

JavaScript Break and Continue: In JavaScript, break and continue are control statements used to manage the flow of loops (for, while, do...while) and switch statements. They allow for more refined control over the iteration process.
Break Statement: The break statement is used to terminate the loop or switch statement entirely. When break is encountered, the control exits from the loop or switch immediately, and the program continues with the next statement outside the loop or switch.
Continue Statement: The continue statement skips the current iteration of the loop and moves directly to the next iteration. Unlike break, continue does not terminate the loop; it just stops the current loop iteration and continues with the next one.
